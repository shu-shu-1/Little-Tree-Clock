# Security Policy

## Supported Versions

We prioritize the security of the latest mainline versions. The scope of security updates is as follows:

| Version | Accepts Security Updates |
| :--- | :--- |
| Main branch & Latest stable release | Yes |
| Older or deprecated preview releases | No |

## Reporting a Vulnerability

We take security very seriously. If you discover any potential severe security vulnerabilities in **Little-Tree-Clock**, **please do not publicly disclose them in a GitHub Issue**.

Please report them via the following method:
1. Send an email to zsxiaoshu@outlook.com.
2. Use `[Security Vulnerability Report] Little-Tree-Clock` as the subject of the email.
3. Provide as detailed reproduction steps as possible, along with the involved environment.

The maintenance team will acknowledge receipt within 72 hours, and assess the risk as soon as possible. If the vulnerability is confirmed, we will work with you to develop a fix plan in the shortest possible time and make an announcement after patching.

Thank you for contributing to the security of this open-source software!
