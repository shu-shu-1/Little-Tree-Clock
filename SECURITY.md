# 安全政策 (Security Policy)

## 支持的版本

我们优先保障最新主线版本的安全性。以下为安全更新的支持范畴：

| 版本号 | 是否接受安全更新 |
| :--- | :--- |
| Main 分支及最新稳定版 | 是 |
| 较旧或者废弃的预览版 | 否 |

## 如何报告安全漏洞

我们非常重视安全性。如果您在 **Little-Tree-Clock** 中发现任何潜在的严重安全漏洞，**请不要在公开的 GitHub Issue 中公开披露它**。

请通过以下方式报告：
1. 发送电子邮件至 zsxiaoshu@outlook.com。
2. 在邮件中使用 `[安全漏洞报告] Little-Tree-Clock` 作为标题。
3. 提供尽可能详细的漏洞重现步骤以及涉及的环境。

维护团队会在收到报告后的 72 小时内确认收到，并尽快对问题进行风险评估。如果是经过确认的漏洞，我们将与您合作，在最短的时间内制定修复计划并在补丁发布后进行公告。

感谢您为保障本开源软件的安全性所作出的付出！
